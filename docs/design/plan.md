### 每日开发流程

```
不要一上来就写代码。每个模块的开发流程：

  1. 想清楚这个模块的输入和输出（5 分钟）
  2. 自己写伪代码或注释骨架（10 分钟）
  3. 自己写实现（主力时间）
  4. 卡在具体技术点时 → 搜文档或问 AI 具体的小问题
  5. 写完后让 AI review 代码
  6. commit + push
```

## 二、风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 已有 Listener 迁移后跑不起来 | 中 | 预留 3/5 一整天排查 |
| 前端 WebSocket 对接困难 | 中 | 先用最简单的消息格式跑通，再优化 |
| CPA/TCPA 算法不确定 | 低 | 公式明确，搜论文或问 AI 确认 |
| 12 天时间不够 | 中 | 砍掉航线模块，优先保证 AIS 通路 + 预警 |

### 如果时间不够的优先级

```
必须完成（汇报底线）：
  ✅ AIS 数据通路跑通（MQTT → 后端 → WebSocket → 前端渲染）
  ✅ 至少一条预警规则可用（CPA < 阈值）

尽量完成：
  ⚠️ 完整预警体系（CPA + TCPA + 安全领域）
  ⚠️ 航线展示

可以延后：
  ⏳ 航线展示（如果时间紧，放到 v0.5 之后补）
```
