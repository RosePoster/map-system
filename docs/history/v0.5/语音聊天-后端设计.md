### 1. Whisper Server API 确认 (whisper.cpp)

**启动命令参考：**
```bash
docker run -d -p 8080:8080 \
  -v ~/workspace/map-system/.runtime/whisper-models:/models \
  ghcr.io/ggml-org/whisper.cpp:main \
  ./server -m /models/ggml-base.bin --host 0.0.0.0 --port 8080
```

**实际接口：**
```http
POST /inference
Content-Type: multipart/form-data

file: <audio binary>
response_format: json (可选，默认 json)
language: zh (可选)
```

**响应格式：**
```json
{
  "text": "转写文本"
}
```

> **注意**：whisper.cpp server 不返回 `confidence`，`BackendChatTranscriptPayload` 中该字段改为可选或移除。

### 8. 技术风险

| 风险 | 缓解措施 |
|------|----------|
| **whisper.cpp server 未启动** | `WhisperClient` 捕获连接异常，返回 `TRANSCRIPTION_FAILED` |
| **音频超大** | `VoiceChatService.validateAudioSize()` 前置校验，返回 `AUDIO_TOO_LARGE` |
| **Base64 解码失败** | try-catch，返回 `INVALID_AUDIO_FORMAT` |
| **转写结果为空** | 检查 `text` 非空非 whitespace，否则返回 `TRANSCRIPTION_FAILED` |
| **WebSocket 帧大小** | 配置 `WebSocketConfig` 增大 buffer（下方补充） |

#### WebSocketConfig 修改建议

```java
@Override
public void configureWebSocketTransport(WebSocketTransportRegistration registry) {
    registry.setMessageSizeLimit(10 * 1024 * 1024);  // 10MB
    registry.setSendBufferSizeLimit(10 * 1024 * 1024);
}
```
