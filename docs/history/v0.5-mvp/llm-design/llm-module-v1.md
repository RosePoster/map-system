# LLM 模块 v1（归档）

> 文档状态：archived / superseded
> 归档日期：2026-04-03
> 说明：本文档保留早期 v1 设计表述，当前实现真值以 `docs/ARCHITECTURE.md`、`docs/EVENT_SCHEMA.md` 为准；后续规划以 `docs/LLM_ROADMAP.md` 为准。

## 1. 版本定位

本文档描述当前已落地的 LLM 最小可交付能力（MVP）。

v1 的定位是：

- 在不改动现有风险计算主链路的前提下，增加可读的风险解释
- 保证主流程稳定，LLM 作为增强能力而非硬依赖
- 形成可上线、可降级、可维护的最小闭环

v1 中 LLM 的职责统一限定为：

- 风险解释
- 规则语义化表达
- 基础场景归类与建议文本生成

LLM 不是控制器，也不是最终决策执行器；不直接下发控制指令，不直接作用于执行层。

---

## 2. 已交付目标

当前版本聚焦以下结果：

- 后端基于现有风险计算结果接入 LLM
- 生成简洁风险解释文本
- 将解释结果接入现有 DTO / API / WebSocket 输出
- 前端完成解释内容渲染

端到端链路为：

结构化风险结果 -> LLM 文本解释 -> 前端展示

说明：当前输出以文本解释为主，面向展示层与决策支持层消费，不构成自动执行指令。

---

## 3. 范围边界

### 3.1 v1 已包含

- 基于现有风险结果构造 LLM 输入
- 调用单一 LLM 接口
- 输出简洁预警解释
- 接入现有输出链路并前端展示

### 3.2 v1 未包含

- 多智能体编排
- 复杂 RAG
- 法规知识库检索
- 自动反思与重试
- 结构化动画指令生成
- 执行层控制指令生成与下发

---

## 4. 设计约束

- 风险计算继续由规则与算法模块负责
- LLM 只负责解释与建议生成，不参与核心物理量计算
- 以最小实现优先，避免提前引入高复杂度能力

最终规则一致性与安全约束责任归属：

- COLREGs 一致性、安全约束与最终控制合法性，不由 LLM 单独保证
- 由确定性规则模块、策略校验层或执行前安全检查模块负责兜底

---

## 5. 接口约束

本版本只约束输入输出方向，不在本文件固化完整字段级 schema。

原因说明：当前实现阶段尚未收敛，不希望过早冻结接口。

后续约束：输入协议将在实现过程中基于真实调用数据进一步收敛，并在后续版本中回写设计文档。

### 输入

来自现有风险计算结果，包含支持解释生成的关键风险信息。

### 输出

面向前端展示的解释内容，形式为：

- 单段文本说明
- 或少量结构化文本块

输出定位：结构化建议输出（文本形态），由展示层与决策支持层消费，不直接下发控制指令。

---

## 6. 集成与触发

- 集成位置：风险计算之后、前端消费之前
- 触发策略：仅在存在风险对象时调用 LLM

---

## 7. 异常与降级

LLM 调用失败、超时或返回异常时，不影响主流程。

可选降级策略：

- 返回空解释
- 返回默认模板解释

异常场景最小约束：

- 调用超时：触发超时保护并直接降级
- 返回异常：解析失败或内容不可用时直接降级
- 主流程保护：不阻塞风险主链路，不影响既有告警输出

---

## 8. 实施顺序（v1）

1. 打通一次最小 LLM 调用
2. 接入现有风险输出链路
3. 前端完成解释展示
4. 补充异常处理与降级

---

## 9. 日志与可观测性建议

建议在 v1 即保留以下日志信息，用于调试、回归分析与问题追踪：

- LLM 原始返回
- 解析结果
- 最终对外输出内容

日志应遵循现有数据治理与脱敏规范。
