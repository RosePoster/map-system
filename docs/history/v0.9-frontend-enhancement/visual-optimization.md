# Step 5：视觉风格精修

> 文档状态：completed
> 最后更新：2026-04-16
> 对齐文档：[`FRONTEND_ENHANCEMENT_PALN.md`](./FRONTEND_ENHANCEMENT_PALN.md)

## 1. 摘要

Step 5 作为 v0.9 的补充视觉步骤，定位为纯表现层调整，不改变 Step 1-4 已完成的功能、协议和交互语义。该步骤已完成，将界面收敛为稳定的“左侧态势监控信息簇、右侧 AI 航海助手侧栏”分区。

已完成的变化：
- **右侧 AI 航海助手**：`RiskExplanationPanel` 调整为全高（100vh）工作区侧栏，宽度增至 380px，更名为“AI 航海助手”，彻底消除了悬浮卡片的视觉割裂。
- **左侧态势监控簇**：`StatusPanel` 与 `TargetsPanel` 实现垂直对齐与视觉收敛，背景、圆角及边框强度完全统一。
- **系统设置集成**：主题切换、语音播报等低频设置并入右侧助手顶部的设置折叠区，主界面保持纯净。
- **周边目标增强**：
    - 为低置信度（<50%）的目标卡片增加了橙色警示提示框。
    - 为目标淡化效果增加了平滑的 `opacity` 过渡动效（300ms ease-in-out）。
- **UI 中文化与精修**：
    - 全面汉化 HUD 标签：DYNAMICS（航行动力学）、PLATFORM（平台系统）、Dist（距离）、TCPA（碰撞时间）。
    - 风险看板标题简化为“风险评估”，消息计数单位更新为“条消息”。
- **展示降噪策略**：仅针对非安全状态（非 SAFE）的目标显示会遇类型（交叉、追越等），显著减少了常态监控下的视觉噪音。
- **对话交互重构**：
    - **画笔图标**：重答编辑按钮替换为气泡右下角悬浮的画笔图标（group-hover 可见）。
    - **扁平化编辑**：取消双层气泡包裹，编辑态直接展示背景半透明的输入框，按钮置于下方。
    - **即时反馈**：点击“重新发送”后，界面立即同步更新消息为“正在处理”状态，消除由于网络延迟导致的视觉停顿感。

## 2. 视觉重构方案

### 2.1 右侧全高工作区侧栏 (AI 航海助手)

- 容器高度扩展为全高，宽度控制在 `380px`。
- 右侧抽屉触发条延伸为全高边缘句柄，配合 `cubic-bezier(0.16, 1, 0.3, 1)` 动效。
- 内部维持三段布局：顶部标题与设置、中部风险评估流水/对话记录、底部 ChatComposer。

### 2.2 左侧态势监控信息簇

- **对齐逻辑**：两个面板沿左侧边缘（left-4）严格对齐。
- **指示器对齐**：连接指示器采用 `grid-cols-[1fr_auto]` 布局，确保 STREAM 与 AI-WS 的状态灯纵向完美对齐。
- **视觉系统**：背景模糊（backdrop-blur-xl）、半透明底色（bg-white/95）和 1px 细边框保持一致。

### 2.3 视觉规范

- **字体**：数值类信息（SOG/COG/HDG/ID）使用等宽字体；标签类文本维持小字号、加宽字距。
- **边框**：统一收敛为 1px 半透明边框（border-white/5 或 border-slate-100）。
- **圆角**：面板容器统一使用 `rounded-xl` 或 `rounded-2xl`，内部子项使用 `rounded-lg`。

## 3. 实施约束

1. **不修改数据流**：仅调整表现层，不改变既有 store 逻辑。
2. **不修改功能**：仅调整样式与交互承载位置。
3. **测试保障**：所有单元测试已根据新 UI 标签（如“航行动力学”、“条消息”）完成同步更新并全部通过。

## 4. 交付总结

该步骤通过侧栏化设计和深度中文化修饰，成功将系统从“分立的监控工具”转化为“一体化的智能航海态势中心”，显著提升了专业感与信息获取效率。
